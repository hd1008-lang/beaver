## Goal

Ensure the beaver repo's own `.claude/` directory reflects the updated harness template. Since beaver dogfoods its own harness, any template change to `src/scaffold/shared/claude-setup.ts` or project-type templates must be propagated into beaver's own `.claude/` by re-rendering and copying the relevant outputs.

## Steps

- [ ] Confirm which files in `.claude/` are generated from the harness template. The beaver repo is its own harness-only install. Key outputs that may change:
  - `.claude/settings.json` — unlikely to change (no new hooks).
  - `.claude/agents/*.md` — unlikely to change (no agent changes).
  - `CLAUDE.md` — **no change needed** for beaver's own `CLAUDE.md` (beaver is a CLI tool, not a user project; its `CLAUDE.md` is handcrafted, not generated by the template). Verify this is hand-maintained by reading the first 20 lines of `CLAUDE.md`.
  - `backlog/README.md`, `plans/README.md` — these are shared templates emitted by `buildClaudeFileMap`. Check if they changed in this feature (they did not — no changes to `buildClaudeFileMap`'s body, only its interface). If unchanged, no sync is needed.

- [ ] Run `npm run build` one final time to confirm the compiled `dist/` is clean.

- [ ] Run `npm run dev` and manually walk through the react-vite flow up to the AI harness prompt. Verify that after selecting an AI option, the `productDescription` prompt appears and a non-empty response is required. (Stop before actual file generation to avoid writing to disk; press Ctrl-C after confirming the prompt fires.)

- [ ] If any `.claude/` files are emitted by `buildClaudeFileMap` and differ from their checked-in counterparts (inspect via `git diff`), copy the re-rendered content into `.claude/`. In this feature, `buildClaudeFileMap`'s body did not change — only its interface — so no `.claude/` file content should differ. Confirm with `git diff --stat` after the build.

- [ ] Archive the plan: move `plans/harness-product-description/` to `plans/.archive/harness-product-description/`. Update all phase frontmatter `status: done` before archiving.

## Verify

```
npm run build                       # exits 0
git diff --stat                     # only src/ and plans/ changes; no unintended .claude/ drift
node .claude/scripts/validate-plans.mjs   # no warnings (all phases done, plan archived)
```

## Notes / Risks

- Per MEMORY.md: "template changes in `src/scaffold/shared/claude-setup.ts` do NOT automatically update the beaver repo's own `.claude/` directory." In this feature, `buildClaudeFileMap`'s body was NOT changed — only its TypeScript interface. The shared emitted files (settings, agents, scripts) are identical. Therefore the sync step is mostly a verification step, not a copy step.
- Beaver's own `CLAUDE.md` is hand-maintained (not generated). Do NOT overwrite it with template output.
- If `git diff` shows unexpected changes, stop and investigate before archiving. Do not archive a plan while unintended drift exists.
- Archive step: `mv plans/harness-product-description plans/.archive/harness-product-description` (dev executes this shell command).
