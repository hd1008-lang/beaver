# docs-writer — Agent Memory

Append durable, non-obvious gotchas and patterns discovered while working — one bullet each, newest first. Link related docs/ files. Read this file at the start of every session.

- **Memory paths now live in neutral `.agents/` root** — Both Claude and Codex harnesses point to `.agents/memory/<agent>/MEMORY.md` (not separate `.claude/` and `.codex/` trees). The spec [[claude-harness.spec.en.md]] was updated 2026-06-22 to reflect this; all 6 references in the Agent Registry, Agent Workflow, and Claude Harness Output sections now correctly document `.agents/memory/`. Guard `scripts/agent-guard-core.mjs` permits writes to `.agents/memory/${agentType}/` only (single unified prefix).
- **Harness choice (Claude/Codex/Both) and Codex scaffold output are now documented** — [[claude-harness.spec.en.md]] extended with three new sections: "Harness Choice Option" (describes the `harness: 'claude'|'codex'|'both'` field), "Claude Harness Output" (tables the .claude/ files), "Codex Harness Output" (tables the .codex/, .agents/, and hook scripts), and "Shared Harness Output" (tables files emitted for any choice). Updated 2026-06-22. Strategy chosen: extend existing spec rather than split into new spec, because both harnesses use the same `buildClaudeFileMap` function and live under a single feature.
- **Product Description field is required in ClaudeHarnessParams** — When Claude Harness is enabled, users must provide a one-line product summary ("Describe your project. What is it?"). It is rendered into CLAUDE.md's `## Project Overview` section for agent onboarding. See [[productdescription_field.md]] for capture/render behavior and [[claude-harness.spec.en.md]] for the full spec (updated 2026-06-21).
- **Agent registry design reverses "no registry" principle deliberately** — For agents specifically, document a minimalist declarative registry (4 fields: name, model, writeScope, memory) to prevent hand-maintained copies drifting across spec/source/runtime. This is stated explicitly in Key Decisions of [[claude-harness.spec.en.md]] (feature spec, not HOW). The reversal is justified because template will host many agents. Keep minimal (4 fields, derivation rules) to avoid becoming a general framework.
- **Backlog IDs are always 4-digit zero-padded** (`backlog/0001`, `backlog/0042`, not `backlog/1`, `backlog/42`). Enforce across [[backlog/README.md]], [[plans/README.md]], and [[claude-harness.spec.en.md]]. Line 38 of plans/README.md already used correct format; line 73 of harness spec had been using 3-digit `<NNN>` which I corrected to 4-digit `<NNNN>`.
- **Keywords must not contain commas or hyphens within single keywords** — the index builder splits on those characters. Use single kebab-case keywords (no hyphenation across words). Example: instead of `menu-flow, project-type`, use `menuflow, projecttype`. Bad characters become separate index entries in "Keyword Index" section of INDEX.md.
