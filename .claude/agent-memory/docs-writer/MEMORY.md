# docs-writer — Agent Memory

Append durable, non-obvious gotchas and patterns discovered while working — one bullet each, newest first. Link related docs/ files. Read this file at the start of every session.

- **Agent registry design reverses "no registry" principle deliberately** — For agents specifically, document a minimalist declarative registry (4 fields: name, model, writeScope, memory) to prevent hand-maintained copies drifting across spec/source/runtime. This is stated explicitly in Key Decisions of [[claude-harness.spec.en.md]] (feature spec, not HOW). The reversal is justified because template will host many agents. Keep minimal (4 fields, derivation rules) to avoid becoming a general framework.
- **Backlog IDs are always 4-digit zero-padded** (`backlog/0001`, `backlog/0042`, not `backlog/1`, `backlog/42`). Enforce across [[backlog/README.md]], [[plans/README.md]], and [[claude-harness.spec.en.md]]. Line 38 of plans/README.md already used correct format; line 73 of harness spec had been using 3-digit `<NNN>` which I corrected to 4-digit `<NNNN>`.
- **Keywords must not contain commas or hyphens within single keywords** — the index builder splits on those characters. Use single kebab-case keywords (no hyphenation across words). Example: instead of `menu-flow, project-type`, use `menuflow, projecttype`. Bad characters become separate index entries in "Keyword Index" section of INDEX.md.
