## Symptom

Running the CI fixture (`test/helpers/scaffold-fixture.ts`, router + zustand + query +
tailwind + biome, harness both, testing on) and then `npm install && npm run build`
inside the scaffolded project fails at the `tsc` step:

```
src/routes/index.tsx(4,38): error TS2345: Argument of type '"/"' is not assignable to parameter of type 'undefined'.
```

`package.json`'s `build` script is `tsc && vite build`. The scaffolded
`src/routes/routeTree.gen.ts` placeholder (written by the scaffold, per
`CLAUDE.md`: "auto-generated by the Vite plugin on first `npm run dev`") has
`// @ts-nocheck` and no `declare module '@tanstack/react-router'` type
registration block, so `createFileRoute('/')` in `src/routes/index.tsx` has
nothing to validate its literal path against, and `tsc` (which runs *before*
`vite build` ever gets a chance to regenerate the tree) fails. `npm run dev`
would work fine since the Vite plugin regenerates the tree with full type
info before the dev server needs types — the break is specific to
`build`'s script ordering when the tree has never been regenerated yet.

## Tried

Reproduced during `plans/assets-and-tests` phase 08 while locally verifying
the weekly-scaffold workflow's local pieces (`node scripts/scaffold-fixture.ts`
→ `npm install` → `npm run build` in a `os.tmpdir()` fixture). `npm install`
succeeds cleanly (pinned versions still resolve, confirmed 2026-07-05). The
`tsc` failure reproduced on the first attempt; not re-run a second time since
it is clearly a template/script-ordering issue, not a flaky install.

Also reproduced on real GitHub Actions 2026-07-05: Weekly Scaffold
`workflow_dispatch` run 28730532593 failed at "Install and build the scaffolded
fixture" with the identical TS2345 error (all prior CI steps green).

## Why parked

Out of scope for phase 08 (CI/workflow wiring only — the plan's non-goals
explicitly exclude changing scaffolded pinned versions or templates). This is
exactly the class of failure `plans/assets-and-tests/08-ci-workflows-and-close.md`
anticipated ("Weekly scaffold WILL eventually fail... that is its job. Its
failure should file a backlog entry, not block push CI") — filing here rather
than fixing the react-vite router template inline.

## Suggested direction

- Likely fix: add a route-tree generation step before `tsc` in the generated
  `build` script (e.g. `"build": "tsr generate && tsc && vite build"` using
  `@tanstack/router-cli`, or reorder to `"vite build && tsc --noEmit"` so Vite's
  plugin regenerates the tree first) — see `src/scaffold/react-vite/templates/package-json.ts`
  and `src/scaffold/react-vite/templates/router.ts`.
- Needs a `dev`-scoped fix touching the react-vite router template; revive via
  `planner` or hand directly to `dev` once prioritized.
